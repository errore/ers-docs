## Introduction

Le système de configuration de style personnalisé vous permet de personnaliser le style de rendu du pipeline de rendu différé d'ERS. Cette fonctionnalité est **uniquement disponible dans le pipeline de rendu différé**.

<Aside type="note" title="Rendu direct">
Si vous utilisez le rendu direct, le pipeline de rendu direct offre une personnalisation plus flexible. Veuillez consulter les nœuds et l'exemple de projet Fab dans la documentation <a href="/fr/nodes/forward-nodes/">Forward Rendering Nodes</a>.
</Aside>

## Styles par défaut (lorsque le style personnalisé est désactivé)

Lorsque **Enable Custom Style** est désactivé, le plugin propose les 4 styles prédéfinis suivants :

| Index de style | Nom du style | Description |
|----------------|---------------|-------------|
| Style 1 | UE Native | Ombrage PBR natif d'UE (diffus Lambert + spéculaire GGX) |
| Style 2 | Adjustable Smooth Toon | Style Toon à lissage adjustable utilisant la transition smoothstep |
| Style 3 | 1-5 Step Toon | Style Toon multi-niveaux avec 1-5 paliers |
| Style 4 | Retro-Reflective | Rétro-réflexion pour créer des effets spéciaux comme les panneaux réfléchissants |

## Configurez votre propre BxDF

Vous pouvez également personnaliser votre propre BxDF en utilisant du code HLSL et les nommer.

Chemin : `Edit → Project Settings → Plugins → Enhanced Rendering Solution`

<Aside type="caution" title="Redémarrage requis">
Vous devez redémarrer l'éditeur pour que les modifications de configuration prennent effet
</Aside>

| Paramètre | Description | Valeur par défaut |
|-----------|-------------|-------------------|
| Enable Custom Style | Activer le style personnalisé | false |
| Style 1 Name | Nom d'affichage du style 1 | Style1 |
| Style 2 Name | Nom d'affichage du style 2 | Style2 |
| Style 3 Name | Nom d'affichage du style 3 | Style3 |
| Style 4 Name | Nom d'affichage du style 4 | Style4 |

### Description des broches

| Broche | Description | Plage |
|--------|-------------|-------|
| Color | Entrée de texture de couleur de base | RGB |
| Data (6bit) | Paramètre de style (voir ci-dessous) | 0~1 |

<Aside type="caution" title="Précision du paramètre Data">
Comme Data utilise un encodage 6bit (précision ~0,016), il est recommandé d'utiliser des algorithmes de tramage (dithering) pour réduire les artefacts de banding de quantification.
</Aside>

### Valeurs recommandées des paramètres de style

| Style | Data recommandé | Description |
|-------|-----------------|-------------|
| Style 1 (UE Native) | 0.5 | Pas d'effet spécial |
| Style 2 (Smooth Toon) | 0.0~1.0 | Plus la valeur est élevée, plus les bords sont lisses |
| Style 3 (Step Toon) | 0.0~1.0 | Plus la valeur est élevée, plus les transitions en paliers sont prononcées |
| Style 4 (Retro-Ref) | 0.0~1.0 | Plus la valeur est élevée, plus le spéculaire est concentré |

<Aside type="tip" title="Fragmentation des ombres Toon">
Si les ombres de l'éclairage Toon semblent trop fragmentées, essayez d'augmenter le paramètre **Contact Shadow Distance** (distance d'ombre de contact) de la lumière principale. Une distance d'ombre de contact plus grande produit des bords d'ombre plus doux et plus continus.
</Aside>

## Personnalisation du Shader (Avancé)

Lorsque les styles personnalisés sont activés, ERS copie les fichiers Shader du répertoire du plugin vers le répertoire de votre projet :

```

```

<Aside type="tip" title="Description du fichier">
`LightStyleDeclare.ush` contient toutes les définitions de fonctions BxDF pour la commutation de style. Vous pouvez modifier ou réécrire la logique de calcul d'éclairage dans ce fichier.

### Signature de fonction et paramètres

Chaque fonction BxDF de style a la signature suivante :

```hlsl
void BxDFStyleN(
    FGBufferData GBuffer,  // Données GBuffer
    half3 N,               // Normale en espace monde
    half3 V,               // Direction de vue
    float Falloff,         // Coefficient de fondu
    half NoL,              // Produit scalaire de la normale et de la direction de la lumière
    FAreaLight AreaLight,  // Données de la lumière
    BxDFContext Context,   // Contexte de calcul auxiliaire
    inout FDirectLighting Lighting,  // Résultat de l'éclairage en sortie
    float Data             // Paramètre de style (0~1)
);
```

### Membres importants de FGBufferData

| Membre | Type | Description |
|--------|------|-------------|
| `WorldNormal` | `half3` | Normale en espace monde (normalisée) |
| `DiffuseColor` | `half3` | Couleur diffuse (0~1) |
| `SpecularColor` | `half3` | Couleur spéculaire (0~1) |
| `BaseColor` | `half3` | Couleur de base |
| `Metallic` | `half` | Métallique (0~1) |
| `Roughness` | `half` | Rugosité (0~1) |
| `Anisotropy` | `half` | Anisotropie (-1~1) |
| `ShadingModelID` | `uint` | ID du modèle d'ombrage |

<Aside type="note" title="Note de version">
Les membres de la structure FGBufferData peuvent varier selon la version d'UE. Veuillez vous référer à votre version réelle du moteur pour la structure exacte.
</Aside>

### Membres importants de BxDFContext

| Membre | Type | Description |
|--------|------|-------------|
| `NoV` | `half` | Cosinus de l'angle entre la normale et la vue |
| `NoL` | `half` | Cosinus de l'angle entre la normale et la lumière |
| `VoL` | `half` | Cosinus de l'angle entre la vue et la lumière |
| `NoH` | `half` | Cosinus de l'angle entre la normale et le demi-vecteur |
| `VoH` | `half` | Cosinus de l'angle entre la vue et le demi-vecteur |

### Exemple de BxDF UE Native par défaut

Ceci est l'implémentation par défaut du Style 1 (UE Native), diffus Lambert standard + spéculaire GGX :

```hlsl
void BxDFStyle1(FGBufferData GBuffer, half3 N, half3 V, float Falloff, 
                half NoL, FAreaLight AreaLight, BxDFContext Context, 
                inout FDirectLighting Lighting, float Data)
{
    // Diffuse
    Lighting.Diffuse = Diffuse_Lambert(GBuffer.DiffuseColor);
    Lighting.Diffuse *= AreaLight.FalloffColor * (Falloff * NoL);

    // Spéculaire
    if (IsRectLight(AreaLight))
    {
        Lighting.Specular = RectGGXApproxLTC(GBuffer.Roughness, GBuffer.SpecularColor, 
                                             N, V, AreaLight.Rect, AreaLight.Texture);
    }
    else
    {
        Lighting.Specular = AreaLight.FalloffColor * (Falloff * NoL) 
                         * SpecularGGX(GBuffer.Roughness, GBuffer.SpecularColor, 
                                       Context, NoL, AreaLight);
    }
}
```

### Implémentation de référence Toon BxDF intégrée

Le code suivant est l'implémentation du style Toon intégré du plugin, pour votre référence et personnalisation :

```hlsl
// Implémentation de référence Toon BxDF
void BxDFStyleToon(FGBufferData GBuffer, half3 N, half3 V, float Falloff, 
                   half NoL, FAreaLight AreaLight, BxDFContext Context, 
                   inout FDirectLighting Lighting, float Data)
{
    // Le paramètre Data contrôle la douceur (0~1)
    float ToonThreshold = 0.5f;
    float ToonSmoothness = Data;

    // Calculer le diffus Toon
    float ToonNoL = smoothstep(ToonThreshold - ToonSmoothness, 
                               ToonThreshold + ToonSmoothness, NoL);

    Lighting.Diffuse = Diffuse_Lambert(GBuffer.DiffuseColor);
    Lighting.Diffuse *= smoothstep(ToonThreshold - ToonSmoothness, 
                                   ToonThreshold + ToonSmoothness, 
                                   AreaLight.FalloffColor) * (Falloff * ToonNoL);

    // Spéculaire Toon
    if (IsRectLight(AreaLight))
    {
        Lighting.Specular = RectGGXApproxLTC(GBuffer.Roughness, GBuffer.SpecularColor, 
                                             N, V, AreaLight.Rect, AreaLight.Texture);
        Lighting.Specular *= ToonNoL;
    }
    else
    {
        float SpecularSize = 1.0 - GBuffer.Roughness;
        float SpecThreshold = pow(Context.NoH, 100.0 * SpecularSize * SpecularSize);
        float ToonSpecular = smoothstep(0.5 - ToonSmoothness, 
                                        0.5 + ToonSmoothness, SpecThreshold);

        Lighting.Specular = AreaLight.FalloffColor * (Falloff * ToonSpecular) 
                          * GBuffer.SpecularColor;
    }
}
```

### Guide de personnalisation

#### 1. Ajuster le seuil diffus

Modifier `ToonThreshold` change la position du bord de l'ombre :

```hlsl
// Seuil plus bas : ombres plus sombres, transition plus précoce
float ToonThreshold = 0.3f;

// Seuil plus élevé : ombres plus claires, transition plus tardive
float ToonThreshold = 0.7f;
```

#### 2. Ajuster la plage de transition douce

Le deuxième paramètre de `smoothstep` contrôle la plage de lissage :

```hlsl
// Plage de lissage plus petite : bords plus durs
float ToonSmoothness = Data * 0.1f;  // Bords nets

// Plage de lissage plus grande : bords plus doux
float ToonSmoothness = Data * 0.4f;  // Transition douce
```

#### 3. Personnaliser la forme spéculaire

Modifier l'exposant de `SpecThreshold` change la forme spéculaire :

```hlsl
// Exposant plus élevé : spéculaire plus concentré, plus petit
float SpecThreshold = pow(Context.NoH, 200.0 * SpecularSize * SpecularSize);

// Exposant plus bas : spéculaire plus grand, plus étalé
float SpecThreshold = pow(Context.NoH, 20.0 * SpecularSize * SpecularSize);
```
