## Overview

The Custom Style Configuration System allows you to customize the rendering style of ERS's deferred rendering pipeline. This feature is **only available in the deferred rendering pipeline**.

<Aside type="note" title="Forward Rendering">
If you are using forward rendering, the forward rendering pipeline offers more flexible rendering customization. Please refer to the nodes and Fab page example project in the <a href="/en/nodes/forward-nodes/">Forward Rendering Nodes</a> documentation.
</Aside>

## Default Styles (When Custom Style is Disabled)

When **Enable Custom Style** is OFF, the plugin provides the following 4 preset styles:

| Style Index | Style Name | Description |
|-------------|------------|-------------|
| Style 1 | UE Native | UE native PBR shading (Lambert diffuse + GGX specular) |
| Style 2 | Adjustable Smooth Toon | Adjustable smoothness Toon style using smoothstep transition |
| Style 3 | 1-5 Step Toon | Multi-step Toon style with 1-5 steps |
| Style 4 | Retro-Reflective | Retro-reflection for creating special effects like reflective signs |

## Configure Your Own BxDF

You can also customize your own BxDF using HLSL code and name them.

Path: `Edit → Project Settings → Plugins → Enhanced Rendering Solution`

<Aside type="caution" title="Requires Restart">
You must restart the editor for configuration changes to take effect
</Aside>

| Parameter | Description | Default |
|-----------|-------------|---------|
| Enable Custom Style | Enable custom styles | false |
| Style 1 Name | Style 1 display name | Style1 |
| Style 2 Name | Style 2 display name | Style2 |
| Style 3 Name | Style 3 display name | Style3 |
| Style 4 Name | Style 4 display name | Style4 |

### Pin Description

| Pin | Description | Range |
|-----|-------------|-------|
| Color | Base color texture input | RGB |
| Data (6bit) | Style parameter (see below) | 0~1 |

<Aside type="caution" title="Data Parameter Precision">
Since Data uses 6bit encoding (precision ~0.016), it is recommended to use dithering algorithms to reduce quantization banding artifacts.
</Aside>

### Style Parameter Recommended Values

| Style | Data Recommended | Description |
|-------|------------------|-------------|
| Style 1 (UE Native) | 0.5 | No special effect |
| Style 2 (Smooth Toon) | 0.0~1.0 | Larger value = smoother edges |
| Style 3 (Step Toon) | 0.0~1.0 | Larger value = more pronounced step transitions |
| Style 4 (Retro-Ref) | 0.0~1.0 | Larger value = more concentrated specular |

<Aside type="tip" title="Toon Lighting Shadow Fragmentation">
If the Toon lighting shadow appears too fragmented, try increasing the main light's **Contact Shadow Distance** parameter. A larger contact shadow distance produces softer, more continuous shadow edges.
</Aside>

## Shader Customization (Advanced)

When custom styles are enabled, ERS copies Shader files from the plugin directory to your project directory:

```
Project/Source/Shaders/ERS/LightStyle/
└── LightStyleDeclare.ush
```

<Aside type="tip" title="File Description">
`LightStyleDeclare.ush` contains all BxDF function definitions for style switching. You can modify or rewrite lighting calculation logic in this file.
</Aside>

### Function Signature and Parameters

Each style's BxDF function has the following signature:

```hlsl
void BxDFStyleN(
    FGBufferData GBuffer,  // GBuffer data
    half3 N,               // World space normal
    half3 V,               // View direction
    float Falloff,         // Falloff coefficient
    half NoL,              // Dot product of normal and light direction
    FAreaLight AreaLight,  // Light data
    BxDFContext Context,   // Auxiliary computation context
    inout FDirectLighting Lighting,  // Output lighting result
    float Data             // Style parameter (0~1)
);
```

### FGBufferData Important Members

| Member | Type | Description |
|--------|------|-------------|
| `WorldNormal` | `half3` | World space normal (normalized) |
| `DiffuseColor` | `half3` | Diffuse color (0~1) |
| `SpecularColor` | `half3` | Specular color (0~1) |
| `BaseColor` | `half3` | Base color |
| `Metallic` | `half` | Metallic (0~1) |
| `Roughness` | `half` | Roughness (0~1) |
| `Anisotropy` | `half` | Anisotropy (-1~1) |
| `ShadingModelID` | `uint` | Shading model ID |

<Aside type="note" title="Version Note">
FGBufferData structure members may vary depending on UE version. Please refer to your actual engine version for the exact structure.
</Aside>

### BxDFContext Important Members

| Member | Type | Description |
|--------|------|-------------|
| `NoV` | `half` | Cosine of angle between normal and view |
| `NoL` | `half` | Cosine of angle between normal and light |
| `VoL` | `half` | Cosine of angle between view and light |
| `NoH` | `half` | Cosine of angle between normal and half vector |
| `VoH` | `half` | Cosine of angle between view and half vector |

### Default UE Native BxDF Example

This is the default implementation for Style 1 (UE Native), standard Lambert diffuse + GGX specular:

```hlsl
void BxDFStyle1(FGBufferData GBuffer, half3 N, half3 V, float Falloff, 
                half NoL, FAreaLight AreaLight, BxDFContext Context, 
                inout FDirectLighting Lighting, float Data)
{
    // Diffuse
    Lighting.Diffuse = Diffuse_Lambert(GBuffer.DiffuseColor);
    Lighting.Diffuse *= AreaLight.FalloffColor * (Falloff * NoL);

    // Specular
    if (IsRectLight(AreaLight))
    {
        Lighting.Specular = RectGGXApproxLTC(GBuffer.Roughness, GBuffer.SpecularColor, 
                                             N, V, AreaLight.Rect, AreaLight.Texture);
    }
    else
    {
        Lighting.Specular = AreaLight.FalloffColor * (Falloff * NoL) 
                         * SpecularGGX(GBuffer.Roughness, GBuffer.SpecularColor, 
                                       Context, NoL, AreaLight);
    }
}
```

### Built-in Toon BxDF Implementation Reference

The following code is the plugin's built-in Toon style implementation for your reference and customization:

```hlsl
// Toon BxDF Reference Implementation
void BxDFStyleToon(FGBufferData GBuffer, half3 N, half3 V, float Falloff, 
                   half NoL, FAreaLight AreaLight, BxDFContext Context, 
                   inout FDirectLighting Lighting, float Data)
{
    // Data parameter controls smoothness (0~1)
    float ToonThreshold = 0.5f;
    float ToonSmoothness = Data;

    // Calculate Toon diffuse
    float ToonNoL = smoothstep(ToonThreshold - ToonSmoothness, 
                               ToonThreshold + ToonSmoothness, NoL);

    Lighting.Diffuse = Diffuse_Lambert(GBuffer.DiffuseColor);
    Lighting.Diffuse *= smoothstep(ToonThreshold - ToonSmoothness, 
                                   ToonThreshold + ToonSmoothness, 
                                   AreaLight.FalloffColor) * (Falloff * ToonNoL);

    // Toon specular
    if (IsRectLight(AreaLight))
    {
        Lighting.Specular = RectGGXApproxLTC(GBuffer.Roughness, GBuffer.SpecularColor, 
                                             N, V, AreaLight.Rect, AreaLight.Texture);
        Lighting.Specular *= ToonNoL;
    }
    else
    {
        float SpecularSize = 1.0 - GBuffer.Roughness;
        float SpecThreshold = pow(Context.NoH, 100.0 * SpecularSize * SpecularSize);
        float ToonSpecular = smoothstep(0.5 - ToonSmoothness, 
                                        0.5 + ToonSmoothness, SpecThreshold);

        Lighting.Specular = AreaLight.FalloffColor * (Falloff * ToonSpecular) 
                          * GBuffer.SpecularColor;
    }
}
```

### Customization Guide

#### 1. Adjusting Diffuse Threshold

Modifying `ToonThreshold` changes the position of the shadow edge:

```hlsl
// Lower threshold: darker shadows, earlier transition
float ToonThreshold = 0.3f;

// Higher threshold: brighter shadows, later transition
float ToonThreshold = 0.7f;
```

#### 2. Adjusting Smooth Transition Range

The second parameter of `smoothstep` controls the smoothness range:

```hlsl
// Smaller smoothness range: harder edges
float ToonSmoothness = Data * 0.1f;  // Sharp edges

// Larger smoothness range: softer edges
float ToonSmoothness = Data * 0.4f;  // Soft transition
```

#### 3. Customizing Specular Shape

Modifying the exponent of `SpecThreshold` changes the specular shape:

```hlsl
// Higher exponent: more concentrated, smaller highlight
float SpecThreshold = pow(Context.NoH, 200.0 * SpecularSize * SpecularSize);

// Lower exponent: larger, more spread out highlight
float SpecThreshold = pow(Context.NoH, 20.0 * SpecularSize * SpecularSize);
```
