## Changelog

<Card title="v1.5.0 - 2026-04-29 | Mobile Support Update">

<details>
<summary>View Update Details</summary>

#### ✨ New Features

##### Mobile Rendering Pipeline Support

- Added mobile rendering pipeline, an optimized rendering solution designed for mobile platforms
- Uses forward rendering mode to achieve efficient stylized rendering effects
- Supports custom HLSL shaders
- Vectorized 4-light processing optimized for mobile GPUs

##### Platform Compatibility

- **Android**: Supports Android 8.0 (API Level 26) and above
- **iOS**: Supports iOS 15 and above
- **VR**: Tested on Pico 4U, theoretically supports all OpenXR-compatible VR devices (e.g., Meta Quest series)

##### Performance Optimization

- Packed data structures to minimize bandwidth consumption
- Stable 60 FPS on mainstream devices
- 90 FPS target framerate on VR devices

---

#### 📚 Documentation

- Added [Mobile Rendering](/en/advanced/mobile/) documentation page
- Detailed mobile HLSL implementation guide
- Performance benchmarks and optimization recommendations

See the [Mobile Rendering Documentation](/en/advanced/mobile/) for more details, including test build downloads and custom HLSL implementation guides.

<Card title="v1.4.0 - 2026-04-01 | Deferred Style Customization Update">

<details>
<summary>View Update Details</summary>

#### ⚠️ BREAKING CHANGES

##### Deferred Main/Auxiliary Light Separation Control

- Main Light and Other Light can now be independently controlled in the deferred rendering path
- This change may affect materials using custom lighting logic in existing projects

<Aside type="caution">
**After upgrading**: Check lighting overlay effects, you may need to fine-tune intensity parameters
</Aside>

---

#### ✨ New Features

##### Custom Style Configuration System

- Supports customizing display names for 4 rendering styles in Project Settings
- Configuration path: `Edit → Project Settings → Plugins → Enhanced Rendering Solution`
- Requires editor restart to take effect
- See [Custom Style Configuration](/en/advanced/custom-style-config/) for detailed documentation

---

#### 🔧 Improvements

##### Brand Upgrade

- Plugin officially renamed to **Enhanced Rendering Solution** (abbreviation ERS unchanged)
- All brand identifiers in documentation and UI have been synchronized

#### 🚀 New Features

##### Multi-Version Unreal Engine Support

- Added support for UE 5.4, UE 5.5

---

#### 🔄 Roadmap Update & Priority Adjustment

Based on community feedback, I'm adjusting development priorities to address a highly-requested feature. Several users have requested expanded lighting style control in deferred rendering, specifically requesting more options beyond the current soft transition defaults. Forward rendering already offers full light/shadow separation with dedicated nodes.

**New priorities:**

- ✅ Delivered: UE 5.4 & 5.5 support
- 🔄 Next: Stylized deferred rendering research - research and implementation begins immediately
- 📱 Later: Mobile support (extended to late April due to increased multi-version maintenance overhead)

**Price update:** As previously communicated, plugin pricing will increase by $10 when mobile support launches. All current users receive all future updates at no additional cost.

Full roadmap: [GitHub](https://github.com/users/errore/projects/1)
